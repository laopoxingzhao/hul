以下是 **hu** 语言的设计选型说明，涵盖语言特性与实现技术两大部分，每项均附选择的理由与权衡。

---

## 一、语言核心设计

### 1. 范式：命令式（Imperative）
- **选择**：语句顺序执行、变量可变、显式控制流。
- **理由**：与主流脚本语言（Python、JavaScript、Lua）一致，降低用户学习成本；实现模型简单，语句可直接映射到解释器循环。

### 2. 类型系统：动态类型（Dynamic Typing）
- **选择**：变量无类型声明，运行时检查操作合法性。
- **理由**：灵活性高，适合快速原型与脚本场景；实现上只需在值上打标签，无需编译期类型推导或检查。

### 3. 内存管理：简单引用计数（`Rc<RefCell<Value>>`）
- **选择**：MVP 阶段所有值通过 `Rc<RefCell<Value>>` 共享，自动释放无引用值。
- **理由**：
  - 避免 GC 的复杂度，快速获得可工作的解释器。
  - `RefCell` 提供内部可变性，支持环境链中的共享更新。
- **已知局限**：循环引用泄漏（如闭包自引用），需要后续引入弱引用或 GC 升级。

### 4. 执行方式：树遍历解释器（直接解释 AST）
- **选择**：无中间表示，Parser 生成 AST 后由 `Interpreter` 直接遍历求值。
- **理由**：MVP 优先实现正确性，避免字节码编译的额外工程；AST 解释足够支持基础语言功能验证。
- **升级路径**：热点分析后可编译为栈式字节码，保持前端不变。

### 5. 语法风格：类 C，分号分隔，大括号定界块
- **选择**：语句以分号结尾，代码块用 `{}` 包裹。
- **理由**：语法无歧义，解析简单（递归下降自然）；符合多数开发者的习惯。
- **假值规则**：只有 `false` 和 `nil` 为假，`0` 和空字符串为真。
- **理由**：避免 `0 == false` 等语言陷阱，使条件判断更精确。

### 6. 宿主：独立运行，REPL + 脚本文件
- **选择**：作为独立二进制，支持交互式 REPL 和读取 `.hu` 文件执行。
- **理由**：无外部依赖，易于测试和分发；REPL 提供即时反馈，方便探索语言。

## 二、实现技术选型

### 1. 实现语言：Rust
- **理由**：高性能、内存安全、强大的 enum 模式匹配非常适合编写解析器和解释器。零成本抽象允许清晰建模 AST 与错误。

### 2. 词法分析：手写 Lexer（迭代器 + 状态机）
- **理由**：零依赖，完全掌控错误位置信息；简单语言的词法规则无需生成器；迭代器模式可惰性产生 token，匹配流式处理。
- **备选**：若语法大幅扩展，可引入 `logos` 声明式词法库。

### 3. 语法分析
- **表达式**：Pratt 解析器（优先级驱动）
  - 理由：通过映射优先级表处理多层运算，扩展新运算符只需增加解析层级，代码规律、易维护。
- **语句**：手写递归下降
  - 理由：语句结构层次分明，递归下降直观映射语言规范，错误恢复可精细控制。

### 4. AST 表示：Rust enum（`Stmt` 与 `Expr` 分离）
- **理由**：类型安全、模式匹配完备，避免非法 AST 节点组合；区分语句与表达式使解释阶段逻辑清晰。

### 5. 中间表示：无（直接解释 AST）
- **理由**：与执行方式一致，省去编译步骤。后续若优化性能，可加入栈式字节码 IR。

### 6. 错误处理：携带源码位置的错误枚举
- **现状**：解析和运行时错误以 `String` 返回，但 `Token` 已保留行/列信息。
- **升级计划**：定义 `Error` 枚举封装阶段、消息和 span，对接 `miette` 或 `ariadne` 输出带箭头的高亮错误。

### 7. 外部工具：暂无
- **备选**：若未来语法规则复杂至难以手写维护，可考虑 `pest` (PEG) 或 `lalrpop` (LR(1)) 作为解析生成器。

## 三、MVP 功能集与对应的设计约束

| 功能 | 选择 | 实现要点 |
|------|------|----------|
| 数据类型 | int, float, string, bool, nil | 统一用 `Value` 枚举，运行时类型检查 |
| 运算符 | 算术、比较、逻辑（and/or/not） | 优先级由 Pratt 层控制，短路逻辑在解释器实现 |
| 变量 | `let` 声明，`=` 赋值，块作用域 | 环境链 + `Rc<RefCell<Environment>>`，赋值沿链查找更新 |
| 控制流 | `if-else`、`while` | 直接条件求值 + 分支/循环递归 |
| 内建语句 | `print` | 硬编码在解释器，输出到 stdout |
| 注释 | 单行 `//` | Lexer 跳过直到换行 |
| 函数 | MVP 后添加，支持闭包 | 将复用环境链实现捕获，值中增加 `Function` 类型 |

---

## 四、选型总结

该选型在 **实现成本**、**可理解性** 与 **扩展灵活性** 之间取得平衡：

- **低实现成本**：手写 lexer/parser + 直接 AST 解释，两周内可从零到完整运行 MVP。
- **高可理解性**：每个组件对应经典编译原理教材模型，易于学习和贡献。
- **明确升级路径**：各模块设计均预留了性能、错误诊断、GC、字节码优化的接口，不会因 MVP 的简化而锁死架构。
